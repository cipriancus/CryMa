1 Abstract

CryMa, short for Cryptocurrency Manager, is an online application with the purpose of managing cryptocurrency information. The website uses the power of semantic web and information found on diverse ontologies to keep the information presented on the site as accurate as possible, with as little as possible human interaction, so that users are updated in real time with the information they need for a better understanding of the world-wide crypto market.

2 Motivation

The world of today grows one's need to adapt to a faster tomorrow, thus it is critical that everybody gets information with as little delay as possible.

Systems capable of displaying diverse information in real time are very sought after.

Smart information visualisation, current and future growth prediction but also sorting and converting capabilities are needed tools for the next financial shift, empowering the people and giving them the oportunity to make the best decisions with the right perspective of the market.

3 Features

The website wishes to provide the following big features :

  • Rich front end client.
  • Top cryptocurrency value : value ( in $ ), change in 24h ( with percentage ), market share and an evolution graphic.
  • The top chart can be ordered ascending or descending for each information in the table header.
  • Machine Learning prediction of future coin values
  • News about cryptocurrencies
  • Extended details about a specific cryptocurrency with information such as : website, logo, trade value, market cap, volume, price evolution chart, markets and news.
  • Mining pool information.
  • Cryptocurrency training for beginners.
  • OpenAPI specification for developers.

Application requirements :

  • Ease of use through intuitive and attractive design of the application.
  • Robustness and functionality.
  • Performance and scalability.
  • Modern approach of the problem.

4 Architecture

In this section the application architecture will be presented, as it is a key application feature.

4.1 Microservices architecture

Each microservice will have a specific function, they will all be called via the GatewayAPI, having the following functions:

  • CoinConversion : Will have data about real currency and cryptocurrency and convert from one another.
  • News : will pull data from the web that are news, related to the cryptocurrencies
  • Plot : using the Prediction microservice, that will predict future coin data based on past information, it's purpose is providing plot data for the front end, based on the filters and preferences set by the latter
  • CoinTology : will store the data about the cryptocurrencies, pull it from the web and manage it so that every service is up to date and has a source of information

8 Back-End

The back-end application will be written in Java using the Spring Boot Framework. Also, as a dependency management tool it was chosen Maven.

A very important part in the back-end is the Apache Jena Framework that has the capabilities for building Semantic Web and Linked Data applications.

From Jena it was chosen to use the following functionalities :

  • RDF API
  • ARQ - SPARQL
  • TDB
  • Fuseki
  • Onthology API
  • Inference API

In order to build a modern application, in the development phase it was chosen a new approach, that of Microservices

Each microservice created will register to a service using the Eureka Service and Spring Cloud.

Each microservice described in the architecture part will serve its sole purpose in the Spring environment, responding to it's REST API, tha API will be helped by a proxy called Netflix Zuul.

In order to have consitency across the application data there are two solutions :

  1. Synchronization via event query using RabbitMQ.
  2. Each microservice will query data via GatewayAPI SPARQL endpoint.

Communication between the GatewayAPI and the microservices will be done via Remote Procedure Invocation ( RPI ). In the back, Spring will convert the url to the url of the microservice and parse the response in order to form the required objects.

Cloud deployment will be done in the CloudFoundry ecosystem, allowing the application to be used across the globe.

9 Modelling And Prediction

9.1 Prediction

The predictions about the monetary value of the cryptocurrencies will be a feature of the application. These predictions will be made by machine learning algorithms based on the historical data available for these coins.

Our approach is based on artificial neural networks.

Since the historical data of a coin is a sequence, the algorithm will use a recurrent neural network, which is specialised for sequences, more specifically, a Long Short Term Memory architecture. Neural networks are a great candidate for this task because they have a great degree of generality(they perform well on unseen data), they’re efficient to use and they have architectures specialised for sequences. The more common approaches for this task used vanilla neural networks, as seen in Stock Market Value Prediction Using Neural Networks and Stock Prediction using Artificial Neural Networks , but the more recent LSTMs outperformed these models and, considering the form of the input, are more fitted for the task and Predicting Stock Prices Using LSTM .
